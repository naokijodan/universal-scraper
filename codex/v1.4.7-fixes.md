## 修正 1: CRITICAL — 既存ユーザーの旧 maxRetry をマイグレーション

### 問題
- 既存ユーザーが過去に `maxRetry: 4` や `5` を保存している場合、現在のコードでは新版でもそのまま温存される
- `initQueueState` (background-queue.js:82-114) は既存 stored.config を spread するだけで clamp していない
- 結果として、リトライが新範囲外の値（4, 5）のまま動き、改修の本来目的（二重送信防止）が破壊される

### 修正対象
**ファイル**: `background-queue.js`
**関数**: `initQueueState` (line 82-114)

### 修正内容
`else` 句（line 95-98）で stored.config を spread した後、`maxRetry` を 0〜3 にクランプする。さらに、stored 値と clamp 後の値が異なる場合は patch に積む。

```diff
       const patch = {};
       if (!Array.isArray(stored[QUEUE_KEYS.queue])) {
         patch[QUEUE_KEYS.queue] = [];
       }
       if (!stored[QUEUE_KEYS.config] || typeof stored[QUEUE_KEYS.config] !== 'object') {
         patch[QUEUE_KEYS.config] = { ...QUEUE_CONFIG_DEFAULTS };
       } else {
         // 既存設定を尊重しつつ、欠けているキーだけデフォルトで補う
-        patch[QUEUE_KEYS.config] = { ...QUEUE_CONFIG_DEFAULTS, ...stored[QUEUE_KEYS.config] };
+        const merged = { ...QUEUE_CONFIG_DEFAULTS, ...stored[QUEUE_KEYS.config] };
+        // §16 (v1.4.7) 旧範囲 (1〜5) からのマイグレーション: 0〜3 に clamp
+        if (typeof merged.maxRetry === 'number') {
+          merged.maxRetry = Math.max(0, Math.min(3, Math.round(merged.maxRetry)));
+        } else {
+          merged.maxRetry = QUEUE_CONFIG_DEFAULTS.maxRetry;
+        }
+        patch[QUEUE_KEYS.config] = merged;
       }
```

## 修正 2: CRITICAL — `markFailedOrRetry` で防御的 clamp

### 問題
- `markFailedOrRetry` (background-queue.js:451-477) は `Number.isInteger(config?.maxRetry)` のみチェックし、範囲チェックしていない
- 何らかの理由で 4, 5 がメモリ内に残った場合に二重送信が再発する可能性
- 多層防御として、ここでも 0〜3 に clamp する

### 修正対象
**ファイル**: `background-queue.js`
**関数**: `markFailedOrRetry` (line 451-477)

### 修正内容
line 452 を以下に変更:

```diff
 function markFailedOrRetry(item, errorMessage, config, now) {
-  const maxRetry = Number.isInteger(config?.maxRetry) ? config.maxRetry : QUEUE_CONFIG_DEFAULTS.maxRetry; // §16 (v1.4.7)
+  // §16 (v1.4.7) 多層防御: 想定外の値が入っても安全側に倒す
+  const rawMaxRetry = Number.isInteger(config?.maxRetry) ? config.maxRetry : QUEUE_CONFIG_DEFAULTS.maxRetry;
+  const maxRetry = Math.max(0, Math.min(3, rawMaxRetry));
   const nextRetryCount = (item.retryCount || 0) + 1;
```

## 修正 3: HIGH — 失敗カードクリック時に `<details>` を自動オープン

### 問題
- 「個別の項目を表示」（`<details id="queue-details">`, options.html:728-734）は `open` 属性を持たず、初期状態は閉じている
- 失敗カードをクリックしてフィルタ状態が変わっても、details が閉じていれば**バッジもリストも見えない**
- 椛島さん視点では「クリックしても何も起こらない」バグになる

### 修正対象
**ファイル**: `options-queue.js`
**関数**: `attachEventListeners` 内、失敗カードのクリックハンドラ (line 575-587)

### 修正内容
クリックハンドラ内で、フィルタを「failed」に切り替えた直後に `details.open = true` を強制する。

```diff
     const failedCard = $('queue-stat-failed-card');
     if (failedCard) {
       failedCard.addEventListener('click', () => {
         // §16 (v1.4.7) 失敗 0 件の全件表示中は無反応にする
         const failedCount = computeStats(state.queue).failed;
         if (failedCount === 0 && state.filterStatus !== 'failed') return;
         state.filterStatus = state.filterStatus === 'failed' ? 'all' : 'failed';
+        // §16 (v1.4.7) フィルタ ON 切替時は details を必ず開く（閉じていたらバッジもリストも見えないため）
+        if (state.filterStatus === 'failed') {
+          const details = document.getElementById('queue-details');
+          if (details && !details.open) details.open = true;
+        }
         renderAll();
       });
+      // §16 (v1.4.7) role="button" tabindex="0" を宣言しているため、Enter/Space でも反応させる
+      failedCard.addEventListener('keydown', (e) => {
+        if (e.key === 'Enter' || e.key === ' ') {
+          e.preventDefault();
+          failedCard.click();
+        }
+      });
     }
```

これで MEDIUM-3（keydown ハンドラ）も同時に対応する。

## 修正 4: MEDIUM — 失敗のみ表示時に件数ヒント

### 問題
- LIST_LIMIT=50 のため、失敗が 50件超のときも先頭 50件しか見えない
- 椛島さんは「失敗のみ」モードで全件確認したいはず
- 隠れた件数があることをUIで明示する

### 修正対象
**ファイル**: `options-queue.js`
**関数**: `renderQueueList` (line 171-198)

### 修正内容
LIST_LIMIT を超える場合は、リスト末尾に件数ヒントを出す。

```diff
   function renderQueueList() {
     const container = $('queue-list');
     if (!container) return;

     // textContent でクリア
     container.textContent = '';

     // §16 (v1.4.7) 失敗カードクリック時は failed のみ表示する
     const rawQueue = Array.isArray(state.queue) ? state.queue : [];
     const filtered = state.filterStatus === 'failed'
       ? rawQueue.filter((q) => q?.status === 'failed')
       : rawQueue;
     const items = filtered.slice(0, LIST_LIMIT);
     renderFilterBadge();

     if (items.length === 0) {
       const empty = document.createElement('div');
       empty.className = 'queue-empty';
       empty.textContent = state.filterStatus === 'failed'
         ? '失敗した項目はありません'
         : 'キューは空です';
       container.appendChild(empty);
       return;
     }

     for (const item of items) {
       container.appendChild(buildQueueItemRow(item));
     }
+
+    // §16 (v1.4.7) 50件超えがある場合のヒント表示
+    if (filtered.length > LIST_LIMIT) {
+      const hint = document.createElement('div');
+      hint.className = 'queue-list-hint';
+      hint.textContent = state.filterStatus === 'failed'
+        ? `失敗 ${filtered.length} 件のうち先頭 ${LIST_LIMIT} 件を表示中`
+        : `${filtered.length} 件のうち先頭 ${LIST_LIMIT} 件を表示中`;
+      container.appendChild(hint);
+    }
   }
```

### 修正対象 (補)
**ファイル**: `options.html`
**位置**: 既存 CSS `#queue-list` 関連の付近（line 501-518 周辺）
