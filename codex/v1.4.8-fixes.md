## 背景

v1.4.8 初版実装に対し、独立レビュー (code-reviewer) と 3者協議 (Claude + GPT-5 + Gemini) で以下が判明:

## 修正 2: HIGH — queueRetryFailed で unknown_remote_state を除外

### 問題
`queueRetryFailed()` (background-queue.js) はサイドパネルの「↻ 失敗をリトライ」ボタンから呼ばれ、status==='failed' の全アイテムを waiting に戻す。これに unknown_remote_state も含まれるため、ユーザーが Sheets を確認せずに一括リトライすると二重送信が再発する。

### 修正内容

**ファイル**: `background-queue.js`
**関数**: `queueRetryFailed()` (現状 line 596-613)

```diff
 async function queueRetryFailed() {
   return withStorageLock(async () => {
     const stored = await chrome.storage.local.get([QUEUE_KEYS.queue]);
     const queue = Array.isArray(stored[QUEUE_KEYS.queue]) ? stored[QUEUE_KEYS.queue] : [];
-    const next = queue.map((q) => q?.status === 'failed'
+    // §17 (v1.4.8) 3者協議結論: unknown_remote_state は一括リトライ対象から除外
+    //              (個別 requeueOne でユーザーが Sheets 確認後にのみ再送可能)
+    let retriedCount = 0;
+    let excludedUnknownCount = 0;
+    const next = queue.map((q) => {
+      if (q?.status !== 'failed') return q;
+      if (q?.lastError === 'unknown_remote_state') {
+        excludedUnknownCount++;
+        return q;
+      }
+      retriedCount++;
+      return { ...q, status: 'waiting', retryCount: 0, nextRetryAt: 0, lastError: null, completedAt: null };
+    });
+    await chrome.storage.local.set({ [QUEUE_KEYS.queue]: next });
+    await writeStats(next);
+    return { success: true, retried: retriedCount, excludedUnknown: excludedUnknownCount };
+  });
+}
-      ? { ...q, status: 'waiting', retryCount: 0, nextRetryAt: 0, lastError: null, completedAt: null }
-      : q);
-    await chrome.storage.local.set({ [QUEUE_KEYS.queue]: next });
-    await writeStats(next);
-    return { success: true };
-  });
-}
```

戻り値で `retried` と `excludedUnknown` の件数を返すことで、UI 側でフィードバックを出せるようにする。

retryFailed() ハンドラで戻り値を使ったフィードバック表示を追加。

## 修正 3: MEDIUM — queueClearFailed で unknown を削除対象外

### 問題
失敗をクリアすると unknown の「Sheets 確認すべき item」のリストが消える。GPT/Gemini ともに「unknown は失敗ログではなく確認待ちの作業リスト」と評価。

### 修正内容

**ファイル**: `background-queue.js`
**関数**: `queueClearFailed()` (現状 line 622-631)

```diff
 async function queueClearFailed() {
   return withStorageLock(async () => {
     const stored = await chrome.storage.local.get([QUEUE_KEYS.queue]);
     const queue = Array.isArray(stored[QUEUE_KEYS.queue]) ? stored[QUEUE_KEYS.queue] : [];
-    const next = queue.filter((q) => q?.status !== 'failed');
+    // §17 (v1.4.8) unknown_remote_state は一括削除対象外（確認待ちリストを守る）
+    let clearedCount = 0;
+    let keptUnknownCount = 0;
+    const next = queue.filter((q) => {
+      if (q?.status !== 'failed') return true;
+      if (q?.lastError === 'unknown_remote_state') {
+        keptUnknownCount++;
+        return true;
+      }
+      clearedCount++;
+      return false;
+    });
     await chrome.storage.local.set({ [QUEUE_KEYS.queue]: next });
     await writeStats(next);
-    return { success: true };
+    return { success: true, cleared: clearedCount, keptUnknown: keptUnknownCount };
   });
 }
```

## 修正 4: MEDIUM — queueRequeueOne で unknown のとき確認ダイアログ

### 問題
個別「↻ 再送」ボタンで unknown を再送するとき、ユーザーが Sheets 確認を済ませているか確認すべき（GPT 推奨）。

### 修正内容

**ファイル**: `options-queue.js`
**関数**: `requeueOne()` (現状 line 504-519 周辺)

```diff
   async function requeueOne(itemId) {
     if (!itemId) return;
+    // §17 (v1.4.8) unknown_remote_state を個別再送する前に Sheets 確認を促す
+    const item = state.queue.find((q) => q?.id === itemId);
+    if (item?.lastError === 'unknown_remote_state') {
+      const ok = confirmAction(
+        '⚠️ この項目は Sheets に書き込まれている可能性があります。\n\n' +
+        '再送する前に Sheets を確認しましたか？\n' +
+        '未確認のまま再送すると重複出品の原因になります。\n\n' +
+        '[OK] 確認済み・再送する\n' +
+        '[キャンセル] まだ確認していない'
+      );
+      if (!ok) return;
+    }
     try {
       const res = await sendMessageSafely({ action: 'queueRequeueOne', itemId });
       if (!res || res.success === false) {
         console.warn('[options-queue] requeueOne failed:', res?.error);
         return;
       }
       await sendMessageSafely({ action: 'kickQueue' });
     } catch (e) {
       console.error('[options-queue] requeueOne error:', e?.message || e);
     }
   }
```
