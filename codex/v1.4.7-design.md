# v1.4.7 設計書 — リトライ既定 0 + 失敗フィルター UI

## ゴール

1. リトライ回数の既定値を 3 → **0**、設定範囲を 1〜5 → **0〜3** に変更
2. 「失敗」ステータスカードをクリックすると、下のリストが「失敗のみ表示」に切り替わる（もう一度クリックで全件表示）
3. バージョンを旧版 → **v1.4.7** に統一

## 背景（なぜこの改修か）

椛島さんの実機観察:
- 20〜30件まとめて送信すると、一度送信が終わったはずの item が「待機」に戻り再送信される
- これは GAS 側で書き込みは成功しているのに、応答受信側で失敗判定 → 自動リトライ → 二重送信が起きるため
- 自動リトライを止めて「失敗」として明示すれば、椛島さんが目視で確認して必要なら手動で再送信できる
- 失敗一覧が混在表示で見にくいので、失敗だけ抽出できるようにしたい

## 守るべき制約（既存コード規約）

- **イミュータブル**: state を直接変更せず、map / filter / spread を使う（既存スタイル）
- **DOM 操作**: `textContent` のみ。`innerHTML` は使わない
- **エラー処理**: 既存の console.error / console.warn パターンを踏襲
- **コメント**: 重要な変更箇所には「§16 (v1.4.7)」のような目印を入れる（既存の §15 (2026-05-13) と同様）
- **イベント結線**: addEventListener、tabindex=0 のキーボード操作は今回スコープ外（必要なら別途）
- **イベントリスナーの二重登録防止**: attachEventListeners は init 内で1回しか呼ばれないので問題なし
